# Release Signal Review

Use this note when deciding whether a project should be described as release-ready.

## Maintenance Checks

- Verify recent release artifacts before mentioning them.
- Prefer repositories with checksums or provenance when supply-chain quality is highlighted.
- Avoid stale version numbers in profile copy.

## Validation

Review the profile README after applying this note. The profile should stay concise, bilingual where needed, and aligned with the current public repository set.
